# Testing

> 全面的测试套件生成 — 单元测试、集成测试、E2E 测试全覆盖

## 使用方法

在 OpenCode 对话中输入 `@testing` 激活此技能。

## 能力

- **单元测试** - 针对函数/组件的边界条件、正常流程、异常流程生成测试
- **集成测试** - 多模块协作场景的测试覆盖
- **E2E 测试** - 用户操作流程的端到端测试
- **测试策略建议** - 根据代码复杂度推荐测试优先级

## 测试框架支持

- Python: pytest, unittest
- JavaScript/TypeScript: Jest, Vitest, Playwright
- Go: testing, testify
- Rust: cargo test

## 触发条件

当用户提到以下关键词时自动激活：
- "写测试"、"测试覆盖"、"单元测试"、"集成测试"
- "TDD"、"测试驱动"、"测一下"

## 示例

用户：`帮我给这个函数写单元测试`
→ 自动生成边界值测试 + 异常处理测试 + Mock 依赖
