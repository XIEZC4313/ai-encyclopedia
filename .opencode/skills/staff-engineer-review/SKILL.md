# Staff-Engineer-Review

> 高级代码审查 — 以 Staff+ 工程师视角审查 PR

## 使用方法

在 OpenCode 对话中输入 `@staff-engineer-review` 激活此技能。

## 审查维度

1. **正确性** - 逻辑是否正确，边界条件是否覆盖
2. **性能** - 时间/空间复杂度，是否有性能瓶颈
3. **安全性** - 注入风险、认证授权、敏感数据处理
4. **可维护性** - 代码结构、命名、复杂度、重复代码
5. **测试覆盖** - 是否有足够的测试覆盖
6. **架构对齐** - 是否符合项目架构和设计规范

## 审查流程

1. 读取变更代码
2. 逐文件分析并记录问题
3. 按严重程度分类（Critical / Major / Minor / Nitpick）
4. 给出具体修复建议和代码示例
5. 综合评分

## 触发条件

当用户提到以下关键词时自动激活：
- "代码审查"、"review"、"review code"、"review this"
- "检查代码"、"看看这段代码"

## 输出格式

```
## 审查报告

### 🔴 Critical
- ...

### 🟡 Major  
- ...

### 🟢 Minor
- ...

### 评分: X/10
```
